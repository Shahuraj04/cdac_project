Role-Based Dashboards (RBAC)

How does the system handle different views for Admin, HR, and Employees?

The core logic lies in Conditional Rendering based on specialized claims in the JWT or session state.

  • Mechanism: Upon login, the user's role is stored in localStorage. The Dashboard.jsx component reads this state.
  • Implementation: Components are swapped dynamically using logical guards: {userData.role === 'ROLE_ADMIN' && <AdminDashboard />}
  • Sidebar Logic: The Sidebar.jsx uses the same role-checking logic to hide/show navigation links, ensuring an Employee never sees the "Employee Management" or "Payroll" links.
Interview Tip: The Security Question "Client-side role checking is great for UI, but I also secured the Spring Boot Backend using @PreAuthorize("hasRole('ADMIN')") on API endpoints to prevent manual URL tampering."

CSV Export Implementation

How did you implement the 'Download Timesheet' feature?

I built a client-side CSV generator using the Web Blob API. This avoids unnecessary server-side file generation.

  • Step 1: Fetch raw attendance data via Axios.
  • Step 2: Format data into a CSV-compliant string (comma-separated values).
  • Step 3: Create a Blob with type text/csv.
  • Step 4: Generate a temporary URL using URL.createObjectURL(blob).
  • Step 5: Create a hidden <a> element, trigger its click(), and immediately revoke the URL to save memory.
Tricky scenario "If a field contains a comma, it could break the CSV. I ensured that each entry is properly joined and sanitized, or for larger datasets, I would use a library like PapaParse."

Custom Hooks Strategy

How did you approach making Hooks like useAuth and useChat?

Hooks were used to extract complex logic from the UI layer to make components reusable and clean.

  • useAuth: Encapsulates localStorage management, JWT token handling, and login/logout state. It provides a single source of truth for the user's session.
  • useChat: Manages the WebSocket (STOMP) lifecycle. It handles connecting, subscribing to private queues, and managing 'is typing' status. By moving this to a hook, the chat logic can be injected into any UI component easily.

General Hooks (The Fundamentals)

What is the difference between useContext and Redux in this project?

I used useContext for lightweight, global state that doesn't change frequently (like the User Profile). For more complex, high-frequency data, local state or external libraries are better.

  • useContext: Solves 'Prop Drilling'. We provide the user data at the top (App level) and consume it anywhere.
  • useEffect: Used extensively for Side Effects, such as fetching DASHBOARD stats as soon as the component mounts or setting up WebSocket listeners.

Advanced "Tricky" Questions

"How do you handle JWT expiration if the user is in the middle of a task?"

Answer: I implemented an Axios Interceptor. It checks for 401 Unauthorized errors in every response. If detected, it automatically calls the logout function and redirects to the login page, protecting sensitive data.

"How does your WebSocket handle a lost connection?"

Answer: The chatService uses an auto-reconnect mechanism. In the connect() callback, if the connection fails, it retries after a short delay, ensuring a seamless user experience for messaging.
